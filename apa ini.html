Executive Summary

Pengembangan aplikasi sistem terintegrasi (ERP) untuk mengatasi kendala aplikasi desktop lama. Proyek mencakup pembangunan Web ERP terpusat, 3 Aplikasi Mobile Android, dan migrasi database. Batasan utama fase ini adalah tidak mencakup integrasi perangkat keras sensor fisik (IoT/SCADA).

Analisa Sistem Berjalan (Current State)

Kendala Utama (Pain Points)

  • Sistem SIA & Billing masih berbasis Desktop kuno.
  • Source Code Terkunci: Menghambat pembaruan sistem dan integrasi dengan database platform lain.
  • Payroll Terbengkalai: Tidak dapat disesuaikan dengan aturan pajak/efisiensi tunjangan terbaru dari pemerintah pusat.

Proses Manual Existing

  • Pencatatan meter pelanggan dilakukan manual via aplikasi lama/foto biasa yang rentan human error.
  • Rekapitulasi penggajian (Payroll) saat ini murni menggunakan Microsoft Excel.
  • Belum ada sistem terpusat untuk pelacakan dan manajemen aset (Zero System).

Detail Kebutuhan Modul Web ERP (Backoffice)

1. Operasional Keuangan & Billing (Fase 1)

Sistem SIA dan Billing lama berbasis desktop masih aktif namun tidak dapat diperbarui. Sistem baru harus mereplikasi dan mengintegrasikan fungsi lama dengan perbaikan:

  • SIA: Mengadopsi struktur Chart of Accounts (COA) lama untuk di-recreate menjadi sistem web yang terintegrasi penuh.
  • Denda: Sistem harus bisa mengeksekusi rumus denda keterlambatan secara dinamis berdasarkan data base pelanggan.

2. Sumber Daya Manusia (HRIS) & Payroll

Aplikasi absensi lama masih digunakan, namun aplikasi payroll lama sudah tidak aktif.

  • Regulasi Pajak: Sistem payroll baru wajib menghitung potongan pajak/tunjangan otomatis sesuai regulasi pemerintah terbaru untuk menggantikan Excel.
  • KPI Karyawan: Seluruh rancangan KPI sedang diperbarui oleh direktur baru dan ditargetkan selesai pada akhir Maret 2026. Pengembangan modul KPI harus menunggu rancangan ini final.

3. Manajemen Aset & Pemeliharaan

Modul ini sepenuhnya baru (belum ada sistem sebelumnya) untuk melacak inventaris dan perawatan terpusat secara digital.

  • Barcode/QR: Sistem web wajib memfasilitasi generate (pembuatan) dan pencetakan QR Code baru untuk ditempel di aset fisik.
  • Preventif: Menyediakan fitur notifikasi/penjadwalan rutin (tahunan/bulanan) untuk servis infrastruktur (seperti maintenance pompa).

Ekosistem Aplikasi Mobile (Android)

Aplikasi 1: Baca Meter (Petugas Lapangan)

  • Wajib Kamera Langsung: Blokir akses galeri, validasi foto sah langsung dari TKP.
  • Fitur OCR: Scan foto angka meteran otomatis menjadi data teks (meminimalisir salah ketik).
  • Offline Mode: Kapabilitas simpan data lokal di blank spot dengan Auto-Sync.

Aplikasi 2: Absensi Karyawan

  • Face Recognition + Liveness: Wajib berkedip/senyum saat absensi untuk cegah kecurangan.
  • Geofencing & GPS: Pengecekan radius kantor dan proteksi aplikasi Fake GPS.
  • Manajemen Shift: Mengakomodasi shift malam, hadir pagi, istirahat, dinas luar, dan cuti.

Aplikasi 3: Manajemen Aset (Tim Teknik)

  • QR Scanner: Pemindaian kode aset fisik untuk melihat detail dan riwayat jadwal servis secara real-time di lapangan.

Pembaruan UI/UX (Modernisasi Desain)

Transisi dari antarmuka aplikasi desktop kaku (Legacy) menuju ekosistem digital modern yang mengutamakan User Experience (UX), responsivitas, dan aksesibilitas pengguna.

💻

Web Dashboard

Implementasi UI Framework modern (Tailwind/Bootstrap). Dashboard analitik tersentralisasi dengan visualisasi data yang bersih dan layout responsif.

📱

Mobile Field-Centric

Desain Touch-Friendly. Tombol besar, warna kontras tinggi untuk kondisi outdoor (silau matahari), dan navigasi minimalis bagi petugas lapangan.

⚡

Frictionless Flow

Pengurangan jumlah klik/langkah pada proses repetitif. Integrasi kamera otomatis untuk OCR dan Absensi guna mempercepat alur kerja harian.

Peta Kompleksitas & Risiko Modul

Timeline Eksekusi Proyek (Gantt Chart)

Estimasi pengerjaan optimal adalah 24 Minggu (6 Bulan). Mengingat kompleksitas, fase perancangan kode inti ditunda hingga ketersediaan data nyata dari pihak PDAM diserahkan paska-kontrak.

Strategi Pengembangan & Rekomendasi

⚙

Pendekatan "No Data, No Code"

Mengingat seluruh struktur payroll baru dan kamus data ditahan hingga kontrak ditandatangani, tim dilarang melakukan pemrograman logika bisnis (terutama Keuangan & SIA) sebelum mendapatkan data tersebut untuk menghindari kerja ulang.

🤖

API Pihak Ketiga untuk Modul Kritis

Sangat tidak disarankan untuk membangun arsitektur Machine Learning secara mandiri. Gunakan layanan Google Vision API / Tesseract untuk fitur pembacaan OCR dan AWS Rekognition untuk fitur Liveness Detection.

📱

Efisiensi Framework & Cross-Platform

Beli dan gunakan Template Admin siap pakai untuk Front-End Web ERP. Untuk pembuatan 3 aplikasi mobile, developer wajib menggunakan framework Cross-Platform (Flutter atau React Native) guna menekan waktu pengerjaan menjadi satu basis kode.
